Sprite

un sprite es un bloque que se puede mover, y que va a recibir eventos. Cada sprite puede tener asignada una ImageMap o una Animation. Si recibe un imageMap pone por defecto el primer cuadro/estado

Constructors

Sprite

(Object customSettings)

Parameters
Object customSettings
Un objeto de inicializacion del Sprite. El objeto debera tener las siguientes propiedades:
src : ImageMap o Animation utilizada por el Sprite
optional int pivotX: el punto utilizado como limite contra el _boundary Layer.
optional int pivotY: idem, tanto para este como para el anterior, si no se especifican toma el centro del Sprite.
optional int class: Un identificador que indica a que clase pertenece. Puedes utilizar algunos de los identificadores predefinidos internamente, o crear indicadores propios
optional speed: la velocidad con la que avanzara... no demasiado util, todavia

public void on

(Event event, Callback callback)

Asigna un evento al Sprite, y su correspondiente callback.

Parameters
Event event
El nombre del evento a asignar. Ademas de los eventos standard (click, keyup, keydown, keypress), tambien incluye 2 eventos no standard:
enterFrame: se ejecuta cada vez que el sprite entra en el cuadro, y previamente al callback principal asignado, y a la actualizacion del frame.
collision: se ejecuta cada vez que el sprite colisiona con otro.
Callback callback
el callback a asignar. Todos los callbacks reciben (eventData, Sprite), que es un objeto con datos, y una referencia al Sprite que tiene asignado el evento. Tambien recibe aCollision, si es collision, que consiste en un array con referencias a Sprites contra los que colisiono.

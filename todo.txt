Class Path:
Un path es una secuencia de puntos, la que se podra interpolar con un tween.
metodos:
    setTween: si se le asigna un tween entonces podra obtener los puntos interpolados (Linear o Bezier)
    requestFrame(): obtiene un frame, si tiene Tween asignado, si no obtiene el siguiente Key
    requestKey(): obtiene el siguiente Key del Path.
    addKeys() toma un array de puntos {x,y, otras propiedades} y lo agrega al array interno de keys... 
        y el resto de los metodos... en realidad va a ser una clase muy parecida a LinearTween, solo que tiene requestFrame, que permite obtener una interpolacion si tiene Tween asignado

Class TileMap: Un TileMap es un objeto que tiene asignado por lo menos un ImageMap, para lo que se ve, y puede ser otro mas exactamente igual para el boundary.
    Podra ser asignar asignado a un Layer, 

    Tambien tiene que tener algun tipo de mapeo, que permita construir los canvas.

    {frame, status, nivelSolidez}
    el nivelSolidez es que tan duro es el objeto, de momento tendra 0 y 255, 0 definira un cuadrado por el que se puede circular, y 255 definira una pared.
